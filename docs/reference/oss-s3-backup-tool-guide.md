## 🎯 시나리오별 최적 도구 추천 (Use Case)

### 1. 기업 자체 서버/DB 백업 및 비용 절감이 최우선일 때

- **추천 도구:** `Kopia` 또는 `Restic`
- **선택 이유:** S3 비용을 가장 크게 아껴주는 **가변 블록 단위 중복 제거(CDC)**
  기술을 탑재했습니다. 랜섬웨어 방어를 위한 `Object Lock` 연동과 강력한
  AES-256 클라이언트 암호화가 기본 제공됩니다.
- **📌 Tip:** 엔지니어가 터미널 스크립트 제어를 선호하면 전통의 **Restic**을,
  마우스 기반의 시점 복구와 빠른 속도를 원하면 **Kopia**를 선택하세요.

### 2. 쿠버네티스(Kubernetes) 클라우드 네이티브 환경일 때

- **추천 도구:** `Velero`
- **선택 이유:** 컨테이너 환경의 백업 표준입니다. 단순 파일 백업을 넘어 K8s의
  클러스터 상태(YAML 설정), 네임스페이스, 퍼시스턴트 볼륨(PV)을 통째로 S3에
  백업하고 다른 클러스터로 원클릭 이관(Migration)할 수 있는 독보적인 기능을
  가집니다.

### 3. GUI 대시보드를 통한 직관적인 관리와 모니터링이 필요할 때

- **추천 도구:** `Duplicati` (데이터 규모 1TB 미만 권장)
- **선택 이유:** 웹 브라우저 기반의 UI가 매우 직관적이며, 스케줄링과 이메일
  알림 설정이 마우스 클릭 몇 번으로 가능합니다. *(단, 데이터 규모가 크다면
  Kopia의 내장 GUI인 KopiaUI를 대안으로 검토하세요.)*

### 4. 원본 구조 그대로 1:1 동기화 및 단순 복사가 필요할 때

- **추천 도구:** `Rclone`
- **선택 이유:** 백업 데이터의 유실 검증이나 압축/중복 제거 프로세스 없이,
  로컬의 결과물을 S3 버킷에 거울처럼 그대로 동기화(Sync)하고자 할 때 가장
  가볍고 빠르게 동작합니다.

## 📜 상용 도입 및 배포 라이선스 가이드

- **사내 내부 운영용 백업인 경우**
  - 본 문서의 6개 도구 모두 라이선스 비용 없이 **100% 무료**로 사내 시스템
    상용 서비스에 사용 가능합니다.
- **자사 솔루션/제품 내부에 백업 기능을 포함(Embed)하여 유료 판매/배포할 경우**
  - **허용 (안전):** `Kopia`, `Restic`, `Velero`, `Duplicati`, `Rclone`
    (MIT, Apache, BSD 계열로 자사 제품 소스코드를 공개할 의무가 없음)
  - ⚠️ **주의 (위험):** `Duplicity` (GPL v2 라이선스로, 소스코드를 연동하여
    제품화할 경우 자사의 상용 솔루션 코드까지 공개해야 하는 법적 리스크 존재)

## 🚀 도입 성공을 위한 Action Item (체크리스트)

- [ ] **1단계: 데이터 성격 파악**
  - 백업할 데이터의 총 용량과 일일 변경량을 파악합니다. 변경량이 많다면
    반드시 **Kopia**나 **Restic**처럼 중복 제거가 강력한 툴을 선택해야 S3
    요금 폭탄을 방지할 수 있습니다.
- [ ] **2단계: S3 Object Lock 활성화**
  - 백업 타겟이 되는 S3 버킷 생성 시 `오브젝트 락(Object Lock)`을 활성화하고,
    백업 툴 설정에서 보존 기간을 연동하여 랜섬웨어 감염 시에도 백업본이 강제
    삭제되지 않도록 방어벽을 구축합니다.
- [ ] **3단계: 모의 복구 시나리오 테스트**
  - 백업 자동화 스크립트 구축 후, 반드시 전체 데이터를 다른 격리된 경로에
    복구(`Restore` 또는 `Mount`)해보는 시뮬레이션을 수행하여 데이터 무결성을
    검증합니다. *(복구되지 않는 백업은 자산이 아니라 부채입니다.)*
