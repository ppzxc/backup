## 2. 100대 규모 인프라 시나리오별 추천 안

### 💡 인프라 현황 진단

- **대상:** 서버 100대분 (자체 접속, 앱, Access, 스위치 로그 등 분류 불가
  대량 로그) + DB 백업 파일
- **핵심 과제:** 실시간으로 발생하는 대량의 잔여 로그 파일로 인한 **API
  요청 비용(PUT) 폭탄 방지**

---

### 🚀 [Option A] 최저 비용 달성 안 (운영 공수 가능 시 추천)

> **"Vector/Fluentd 번들링 아키텍처 + AWS S3 Glacier Deep Archive"**

- **구조:**
  1. 100대의 서버 및 스위치 로그를 중간 **로그 취합 서버(Log Aggregator)**로
     실시간 전송 (Vector 또는 Fluent-bit 활용).
  2. 취합 서버에서 로그 분류 없이 **일별 또는 시간별로 통짜 압축
     파일(`.tar.gz`)**로 변환.
  3. 압축된 대형 파일만 **AWS S3 Glacier Deep Archive**로 업로드.
- **비용 절감 원리:** 수천만 건의 로그를 단 몇 개의 파일로 묶어서 업로드함으로써,
  Glacier의 치명적인 약점인 **비싼 PUT API 비용을 무력화**하고 전 세계 최저
  수준의 저장 단가 혜택만 누림.
- **장점:** 10TB 기준 월 2~3만 원 대의 압도적인 최저 비용 달성.
- **단점:** 중간 취합 파이프라인 관리 공수 필요, 재난 복구 시 로그 다운로드에
  12~48시간 대기 필요.

---

### 🚀 [Option B] 관리 편의성 극대화 안 (인프라 관리 최소화)

> **"직접 전송 아키텍처 + Wasabi Hot Cloud Storage"**

- **구조:**
  1. 중간 취합 서버 없이, 100대의 서버에서 각자 주기적(예: 5분~10분 단위)으로
     로그를 잘라 **Wasabi 버킷으로 직접 업로드**.
- **비용 절감 원리:** Wasabi는 **API 요청 비용이 0원($0.00)**이므로 100대의
  서버가 개별적으로 자주 업로드해도 API 요금 폭탄 걱정이 없음.
